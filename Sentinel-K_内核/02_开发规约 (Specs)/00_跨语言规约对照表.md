# 00_跨语言规约对照表 (Cross-Language Specs)

> **定位**: Sentinel-K 跨语言适配指南。当指挥官切换开发语言时，应参照下表重置 AI 的 **Vibe** 环境。

## 📊 核心规约对照矩阵

| 维度           | **Java (Spring/RuoYi)**          | **Python (FastAPI/AI)**          | **Go (Gin/Cloud-Native)**                  | **Value Core (不变项)** |
| :------------- | :------------------------------- | :------------------------------- | :----------------------------------------- | :---------------------- |
| **行业标准**   | **阿里开发手册 (泰山版)**        | **Google Style / PEP 8**         | **Uber Go Style Guide**                    | **业界顶级共识**        |
| **风格检测**   | Checkstyle / P3C                 | **Ruff** / Flake8                | **golangci-lint**                          | **自动化守卫**          |
| **强类型检查** | (原生支持)                       | **Mypy** / Pyright               | (原生支持)                                 | **契约确定性**          |
| **测试框架**   | JUnit 5 / AssertJ                | **pytest**                       | `go test` / Testify                        | **机器见证 (Evidence)** |

## 🛠️ Sentinel-K 适配建议

### 1. Python 适配 (Vibe: 极速严谨)
- **AI 调教关键**: 强制要求 AI 使用 **Type Hints** 和 **Pydantic V2** 作为 `Bone` (骨架层)。
- **Evidence 门槛**: `Ruff check` 必须 0 告警，`pytest` 覆盖核心分支。
- **Solid Assets**: `pyproject.toml`, `requirements.txt`, `alembic` (Migration)。

### 2. Go 适配 (Vibe: 简洁高效)
- **AI 调教关键**: 强制 **Interface** 驱动设计。要求 AI 在 Evidence 中展示 `golangci-lint` 的摘要。
- **Evidence 门槛**: 遵循 `Uber Style` 的错误处理（`if err != nil` 必须处理，严禁忽略）。
- **Solid Assets**: `go.mod`, `api/*.proto` (契约文件), `internal/` (封印私有逻辑)。

### 3. Java 适配 (Vibe: 工业级稳定)
- **AI 调教关键**: 遵循 **阿里规约**。强制 `DTO/VO` 物理隔离。
- **Evidence 门槛**: 严格执行 `[K-PIZZA]` 切片。
- **Solid Assets**: `pom.xml`, `SQL Migration`, `SecurityConfig`。
