# 项目全景图 (Landscape)

> 此文件是 AI 理解项目的核心入口。请保持更新。

## 1. 项目元数据
- **项目名称**: Sentinel-K (Sentinel Kernel)
- **核心目标**: 构建一套专为 AI 与人类高效协同设计的“协作操作系统”，实现工程级的受控执行、审计与资产管理。
- **技术栈**: 
  - **核心协议**: 基于 GitHub Flavored Markdown 的工作流指令集。
  - **分层结构**: 内核层 (Kernel) + 工作流层 (Workflows) + 执行层 (Body) + 归档层 (Archive)。
  - **协同工具**: 深度集成 AI Agent (执行官) 与 人类 (指挥官) 的交互闭环。
- **架构模式**: 意图驱动 (Intent-Driven) -> 原子执行 (Atomic Execution) -> 证据验证 (Evidence Verification)。

## 2. 核心业务全景
- **内核控制区 (Kernel Layer)**:
  - **SSOT (Single Source of Truth)**: `00_系统总纲` 定义最高宪法。
  - **资产分级**: 实现 `Solid` (强管制) 与 `Liquid` (流动) 资产的分类治理。
  - **启动自检**: 通过 `Pack-0/1/2` 等分层数据包加载策略，物理隔离幻觉。
- **自动化工作流 (Workflow Layer)**:
  - **全生命周期 SOP**: 包含 `/01_启动自检`、`/02_需求雷达`、`/03_受控执行`、`/04_质量关卡`、`/05_归档重启`。
  - **反幻觉自检**: 内置自评模型与信心评分卡 (`[K-SCORE]`)。
- **执行与验证 (Body & Evidence)**:
  - **切披萨法则 (`[K-PIZZA]`)**: 强制复杂任务拆解为 Bone/Stub/Muscle 阶段。
  - **证据链驱动**: 交付物必须附带 `Verification Evidence`，确保逻辑可追溯。

## 3. 当前里程碑 (Current Milestone)
- **阶段**: 内核 2.0 工业化对齐
- **重点**: 完善多语言规约对照表，强化 Pizza Logic 的物理执行，提升证据块的机器可读性。

## 4. 已知风险与技术债 (Known Issues)
- **上下文膨胀**: 随着项目复杂度增加，Pack 加载需要更加精准（引入 Lazy Loading 机制）。
- **跨语言适配**: Python/Go 等语言的规约细节仍在补全中（参考 `02_开发规约`）。
- **幻觉噪声**: 长期会话中 AI 可能漂移出总纲约束，需定期执行 `/05_归档重启`。
