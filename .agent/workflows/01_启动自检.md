# 🟢 01_启动自检 (System Boot V2.0)

> **定位**: 每日开工或物理重启会话时的“自检协议”。
> **目的**: 物理隔离历史幻觉，建立“清醒”的执行环境。
> **V2升级**: 集成 `Context Pack` 加载策略与 `Self-Confidence` 评分卡。

## Node 0: 语言协议 (Language Protocol)

> **最高指令**: 本次会话中，所有输出必须严格遵守以下语言规范：
1.  **Artifacts (工件)**: `walkthrough.md`、`implementation_plan.md`、`task.md` 等所有生成的文档文件，必须使用 **中文 (Chinese)** 撰写。
2.  **对话 (Conversation)**: 与用户的交流、思考过程、汇报总结，必须使用 **中文 (Chinese)**。
3.  **例外**: 代码变量名、日志关键字、错误堆栈保持英文原样。

## Node A: 内核加载 (Kernel Load) - Pack-0

> 执行动作:
1.  **加载 Pack-0 (System Kernel) `[K-BOOT]`**: 
    - 读取 `Sentinel-K_内核/00_系统总纲 (Kernel).md` (最高宪法)。
    - 读取 `Sentinel-K_内核/05_项目进度 (Tasks).md` (了解全局进度)。
    - 读取 `.cursorrules` (若有) 或项目根目录核心引导。
2.  **加载 Pack-1 (Bone - Arch) `[K-BOOT]`**:
    - 读取 `Sentinel-K_内核/01_项目全景 (Landscape).md` (技术栈与模块边界)。
    - **按需检索** `Sentinel-K_内核/02_开发规约 (Specs)/`：AI 应使用 `grep` 检索相关规约段落，严禁盲目全量加载。
    - 读取 `Sentinel-K_内核/06_业务拓扑 (Topology).md` (上下游依赖)。

## Node B: 信心自评 (Self-Confidence Check)

请执行以下指令进行自我诊断，并输出 **Confidence Score Card**：

1.  **反面案例验证 (Anti-Pattern Validation)**:
    *   > "依据 `01_项目全景` 或 `02_开发规约` 定义的包结构/模块约束，请构造一个**违反项目规范**的类路径或依赖示例，并解释其非法原因。"
    *   *(目的: 证明 AI 已建立项目特定的“禁区”意识)*
2.  **技术栈对齐 (Stack Alignment)**:
    *   > "确认本项目关于持久层、JSON序列化及核心算法库的具体选型，严禁带入其他项目的惯性依赖。"
3.  **信心打分 (Confidence Score) `[K-SCORE]`**:
    *   > "输出符合内核 `[K-SCORE]` 规范的 **Confidence Score Card**，并严格执行熔断/警告协议。"

## Node C: 决策点 (Decision Point)

向用户汇报：
> "指挥官，自检完成。
> *   反面验证: [通过/失败]
> *   技术栈: [从 `01_项目全景 (Landscape).md` 提取并复述]
> *   信心评分: [分数]
> *   盲区: [无/列表]
>
> 请输入 **ACK** 进入下一阶段 (`02_需求雷达`)，或输入 **NACK** 要求补充上下文。"
